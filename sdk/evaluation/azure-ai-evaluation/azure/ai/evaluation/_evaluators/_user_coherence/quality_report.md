# User Coherence Evaluator — Quality Report

## Experiment Setup

### Objective

Validate the coherence of the user simulation using MCS's Coherence metric. The evaluator assesses **user coherence** (derail detection) — whether each user turn in a conversation is a natural, logical follow-up to the preceding assistant response and overall conversation history.

### Model Configuration

- **Judge model**: `gpt-4.1` (Azure OpenAI)
- **Temperature**: 0.0 (deterministic)
- **Response format**: JSON

### Data Source

Conversations were fetched from two Azure AI Foundry agents using `fetch_agent_conversations.py`, which retrieves messages via the Foundry `conversations.items.list()` API and saves them in chronological order.

| Agent | Domain | Conversations |
|-------|--------|:------------:|
| **legal-contract-review-sim** | Legal contract review (SaaS agreements, liability clauses) | 50 |
| **airline-customer-service-sim** | Airline customer service (refunds, delays, rebooking) | 50 |

**Total conversations evaluated**: 100

---

## What the Evaluator Evaluates

The Coherence Metric by MCS performs **per-step derail detection** on multi-turn conversations:

1. **Per-step scoring**: For each user turn, the evaluator determines whether the user's message is a logical follow-up given the previous assistant response and the full conversation history.

2. **Scoring rubric** (0–2 scale):
   | Score | Label | Meaning |
   |:-----:|-------|---------|
   | **2** | Likely and logical | The user question is a natural, logical response a real user would likely write. |
   | **1** | Weak but possible | The question is a somewhat weak or unlikely follow-up, but a user could potentially respond this way. |
   | **0** | Not logical | There is no logical way a real user would respond this way; the question appears disconnected or unrelated. |
   | **null** | N/A | First turn (no prior assistant response to compare against). |

3. **Aggregate score**: The conversation-level `user_coherence` score is the **mean** of all per-step `derail_score` values (excluding nulls).

4. **Per-step output**: Each step includes a `conversation_step` index, `derail_score`, and a short natural-language `explanation`.

## Key Findings

1. **Perfect coherence across all conversations**: Every single user turn across all 100 conversations and 438 scored steps received the maximum score of 2 (Likely/logical). No derailed or weakly-connected turns were detected.

2. **Consistent across domains**: The evaluator produced identical results for both a professional/legal domain and a consumer/airline domain, suggesting the scoring is robust across conversation styles.

3. **Expected result for simulation data**: These conversations were generated by well-designed Foundry agent simulations where user messages are crafted to follow a coherent narrative. The perfect scores are the expected outcome — the simulated users don't exhibit derailed behavior.

4. **Evaluator sensitivity remains untested on these data**: Because the simulated conversations contain no derailed turns, this experiment validates that the evaluator correctly identifies coherent conversations but does **not** test its ability to detect actual derailment ('Score 0' or 'Score 1' cases). Testing with adversarial or intentionally-derailed conversations is needed to validate sensitivity.

5. **Zero variance**: The standard deviation of 0.000 across all conversations confirms uniform behavior but also means these data provide no signal for discriminating between conversation quality levels.

---

## Recommendations for Follow-Up

- **Test with adversarial data**: Introduce conversations with intentionally injected off-topic or nonsensical user turns to validate that the evaluator correctly assigns scores of 0 or 1.
- **Test with real (non-simulated) user data**: Real user conversations may exhibit more varied coherence patterns.
- **Cross-model comparison**: Evaluate with different judge models to assess scoring consistency.
